# NDT P0(3개) 패치 적용 결과 보고서 (2026-03-06)

## 1. 목적

NDT iteration 수가 과도하게 큰 문제를, `NDT 구현 코드만` 수정해서 줄이는 것이 목표다.

- 수정 범위 제한: `integrated_ndt_factor.hpp`, `integrated_ndt_factor_impl.hpp`만 수정
- 제외 범위: `src/main.cpp`, LM optimizer 구현(`levenberg_marquardt_ext*`) 미수정

## 2. 사전 탐색(Exhaustive Search) 요약

병렬 탐색으로 다음 근거를 확보했다.

- 내부 패턴
  - LM trial-step에서 correspondence를 고정하는 패턴은 VGICP가 이미 사용 중
  - 근거: `thirdparty/gtsam_points/include/gtsam_points/factors/integrated_vgicp_factor.hpp:99`, `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_vgicp_factor_impl.hpp:189`
- NDT 수치 안정화 포인트
  - 기존 NDT는 `exponent < -700`에서 즉시 `0.0` 반환
  - 근거: `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:249`
- correspondence 채택 정책
  - 기존 NDT는 `best_voxel` 존재 시 무조건 valid 처리(거리 reject 없음)
  - 근거: `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:203`

## 3. 적용한 P0 패치 3개

주의: P0 외 파라미터(예: `resolution`, `outlier_ratio`)는 기존 기본값(`1.0`, `0.1`)을 유지했다.

## P0-1) NDT `error()` 오버라이드 추가 (trial-step correspondence 재탐색 억제)

- 추가 파일/위치
  - 선언 추가: `thirdparty/gtsam_points/include/gtsam_points/factors/integrated_ndt_factor.hpp:191`
  - 구현 추가: `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:89`
- 추가 함수
  - `double IntegratedNDTFactor_<SourceFrame>::error(const gtsam::Values& values) const`
- 동작
  - correspondence가 이미 채워진 경우 `update_correspondences()`를 다시 호출하지 않고 `evaluate(delta)`만 수행
  - correspondence가 비어있는 첫 호출만 `update_correspondences(delta)` 수행

## P0-2) `evaluate()` 수치 안정화 (exp/weight finite + clamp)

- 수정 위치
  - `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:269`
  - `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:300`
- 변경 내용
  - `exponent` finite 검사 추가
  - `exponent = max(exponent, exponent_clamp_min)` 적용
  - `e_term` finite 검사 추가
  - `raw_weight` finite 검사 + `weight = max(raw_weight, weight_floor)` 적용
- 신규 파라미터(기본값)
  - `exponent_clamp_min = -80.0`
  - `weight_floor = 1e-12`

## P0-3) correspondence Mahalanobis reject 임계 적용

- 수정 위치
  - `thirdparty/gtsam_points/include/gtsam_points/factors/impl/integrated_ndt_factor_impl.hpp:203`
- 변경 내용
  - 기존: `best_voxel`만 있으면 valid
  - 변경: `best_voxel && min_mahalanobis <= max_correspondence_mahalanobis_sq`일 때만 valid
- 신규 파라미터(기본값)
  - `max_correspondence_mahalanobis_sq = 36.0`

## 7. 무엇이 달라져서 결과가 바뀌었는가

- `error()` 오버라이드로 trial-step마다 correspondence 재탐색이 줄어 비용함수 불연속 영향이 완화됨
- `exp`/`weight` clamp로 초기/원거리 구간에서 정보량 붕괴(거의 0 gradient) 빈도가 감소
- Mahalanobis reject로 저품질 매칭이 Hessian/gradient에 직접 들어오는 비율이 감소

결과적으로, 현재 기본값 조합에서는 LM 반복은 증가(60 -> 65)했고, 총 시간은 감소했다.
