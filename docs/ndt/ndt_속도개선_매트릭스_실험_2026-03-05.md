# NDT 속도 개선 매트릭스 실험 (2026-03-05)

> 참고: 본 문서는 NDT 내부 파라미터를 일시적으로 스윕한 실험 보고서입니다. 현재 `main` 코드는 비교 공정성 정책에 따라 NDT 전용 외부 조절 경로를 제거했고, `DIRECT7` + `outlier_ratio=0.01` 고정으로 동작합니다.

## 실험 목적

- TODO 1번 항목인 "NDT 속도 개선 실험 매트릭스 추가 실행" 수행
- NDT에서 `search_mode`, `outlier_ratio`, `threads`, `correspondence_update_tolerance`가 속도/정확도에 주는 영향을 계량 비교

## 실행 환경 및 커맨드

- 실행 바이너리: `build_local/lidar_registration_benchmark`
- 실행 방식: `--headless --factor NDT`
- 스윕 축
  - `search_mode`: `DIRECT1`, `DIRECT7`, `DIRECT27`
  - `outlier_ratio`: `0.01`, `0.05`, `0.1`
  - `threads`: `1`, `12`, `24`
  - `corr tolerance`: `(rot, trans) = (0.0, 0.0)`, `(0.005, 0.02)`
- 총 실행 수: `3 x 3 x 3 x 2 = 54`

## 기준선 (baseline)

- 설정: `DIRECT7`, `outlier=0.01`, `threads=24`, `corr=(0.0,0.0)`
- 결과: `Mean T=0.097598 m`, `Mean R=0.714911 deg`, `Time=3478.067 ms`

## 핵심 결과

### 1) 정확도 10% 이내 조건에서 최고 속도

- 후보 조건: `Mean T <= baseline*1.10` AND `Mean R <= baseline*1.10`
- 최적 설정:
  - `DIRECT7`, `outlier=0.01`, `threads=12`, `corr=(0.005,0.02)`
  - `Mean T=0.102064 m` (`+4.58%`)
  - `Mean R=0.680913 deg` (`-4.76%`, 오히려 개선)
  - `Time=1901.169 ms` (`-45.34%`, 약 `1.83x` 빠름)

### 2) 전체 최속 설정 (정확도 제약 없음)

- 최속: `DIRECT1`, `outlier=0.01`, `threads=12`, `corr=(0.005,0.02)`
- 결과: `Mean T=0.198480 m`, `Mean R=1.458652 deg`, `Time=520.798 ms`
- 해석: 속도는 매우 빠르지만 정확도 손실이 커서 고정밀 정합에는 부적합

### 3) search_mode 경향

- `DIRECT1`: 가장 빠름, 정확도 손실 큼
- `DIRECT7`: 속도/정확도 균형이 가장 좋음
- `DIRECT27`: 정확도는 `DIRECT7`과 비슷하거나 일부 구간에서 미세 개선, 시간은 크게 증가

### 4) 스레드 스케일링 (대표: DIRECT7, outlier=0.01, corr=(0,0))

- `threads=1`: `21016.759 ms`
- `threads=12`: `3846.885 ms`
- `threads=24`: `3478.067 ms`

해석:
- 1 -> 12는 큰 이득 (`~5.46x`)
- 12 -> 24는 추가 이득이 작음 (`~1.11x`)

## 결론

- 현재 데이터/코드 기준의 실무 권장값은 다음과 같다.
  - `search_mode = DIRECT7`
  - `outlier_ratio = 0.01`
  - `threads = 12` (24 대비 효율 대비 이득이 더 좋음)
  - `correspondence_update_tolerance = (0.005, 0.02)`
- 위 조합은 baseline 대비 정확도 손실을 10% 이내로 유지하면서 시간은 약 45% 단축했다.
