# MNDT (Mixture NDT) 구현 플랜 상세 보고서

## 0. 문서 목적

이 문서는 Bottom-LiDAR 코드베이스에 `MNDT(GMM-NDT)`를 도입하기 위한 구현 계획을 정리한다.

- 범위: 계획 수립만 (코드 변경 없음)
- 목표: 기존 NDT의 hard assignment 한계를 완화하면서 LM 안정성을 유지
- 핵심 원칙: `E-step(책임도 갱신)`과 `M-step(LM 최적화)`를 분리한 MM/EM 스타일 적용

## 2. 수학적 설계 원칙

## 2.1 표기

- 소스 포인트: `x_i`
- pose 파라미터: `theta`
- 변환된 포인트: `q_i(theta) = T(theta) x_i`
- 혼합 성분 `k`: 평균 `mu_k`, 공분산 `Sigma_k`, 가중치 `pi_k`
- 정밀도 행렬: `Lambda_k = Sigma_k^{-1}`

## 2.2 Exact mixture 목적함수 (모니터링용)

MNDT의 원래 음의 로그우도는 다음과 같다.

```math
s_i(theta) = pi_0 c_0 + \sum_{k \in K_i} pi_k \mathcal{N}(q_i(theta); mu_k, Sigma_k)
```

```math
F(theta) = -\sum_i \log s_i(theta)
```

- `pi_0 c_0`: uniform outlier component (권장)
- `K_i`: 포인트 `i`의 top-K 후보 성분 집합

이 식 자체를 그대로 LM에 넣으면 `gamma(theta)` 의존성까지 포함되어 곡률이 불안정해질 수 있다.

## 2.3 E-step: 책임도(gamma) 계산

수치 안정성을 위해 log-space에서 계산한다.

```math
a_{ik} = \log pi_k - \frac{1}{2}\left(d_{ik}^2 + \log |2\pi Sigma_k|\right)
```

```math
d_{ik}^2 = r_{ik}^\top Lambda_k r_{ik}, \quad r_{ik}=q_i(theta^t)-mu_k
```

```math
\log s_i = \mathrm{logsumexp}(\log(pi_0 c_0), \{a_{ik}\}_{k \in K_i})
```

```math
gamma_{ik} = \exp(a_{ik} - \log s_i)
```

후처리:

- `gamma_{ik} <- max(gamma_{ik}, gamma_min)`
- 정규화 재수행

## 2.4 M-step: LM 친화형 surrogate

LM에서는 `gamma`를 고정한 surrogate를 최소화한다.

```math
Q(theta \mid gamma^t)=\frac{1}{2}\sum_i\sum_{k\in K_i} gamma_{ik}^t\, r_{ik}(theta)^\top Lambda_k r_{ik}(theta)
```

SE(3) 미소 업데이트에서 Jacobian을 `J_i`라 두면:

```math
g = \sum_{i,k} gamma_{ik}^t J_i^\top Lambda_k r_{ik}
```

```math
H \approx \sum_{i,k} gamma_{ik}^t J_i^\top Lambda_k J_i
```

```math
(H + \lambda I)\,\delta = -g
```

### PSD 근거

각 항 `gamma * J^T * Lambda * J`는

- `gamma >= 0`
- `Lambda = Sigma^{-1} \succ 0`

이면 PSD다. 따라서 H1-only 근사에서는 Hessian이 PSD 성질을 유지한다.

## 3. 알고리즘 스케줄 (중요)

`outer`와 `inner`를 분리한다.

1. `theta^t`에서 top-K 후보 및 `gamma^t` 계산 (E-step)
2. `gamma^t` 고정 상태로 LM inner solve 수행 (M-step)
3. 수용된 새 포즈 `theta^{t+1}`를 얻으면 다음 outer로 진행
4. outer마다만 mixture 재계산

금지 사항:

- LM inner damping 루프 도중 `gamma`/candidate 재계산
- `error()`와 `linearize()`가 서로 다른 mixture cache를 사용하는 것

## 5. 실패 모드와 수학적 대응

## 5.1 Inner reject 폭증

원인:

- LM trial마다 `gamma`/candidate가 변해 모델 충실도 붕괴

대응:

- `Q(theta|gamma^t)` 고정 후 inner solve
- outer accept 후에만 `gamma` 갱신

## 5.2 비정상 곡률/불안정 선형계

원인:

- `Sigma` 근사 singular
- `gamma(theta)` 미분항까지 섞어 비선형 곡률 악화

대응:

- H1-only 유지
- `Sigma` eigen floor
- LM damping 하한 유지

## 5.3 outlier 지배

원인:

- mixture 우도 분모가 작아져 `gamma` 극단화

대응:

- uniform outlier component
- `gamma_min` 및 `d2_max` gating

## 7. 이번 계획의 채택 근거 요약

1. 현재 코드의 `error/linearize` 이중 업데이트 구조에서는 mixture를 inner에서 갱신하면 LM 불안정성이 높다.
2. `gamma` 고정 surrogate `Q`는 weighted least-squares 형태라 현재 H1-only NDT 아키텍처와 결합이 쉽다.
3. K=1 parity 선행은 회귀를 막는 최소 안전장치다.
4. K=2 이상은 표현력 이점이 있으나 계산량/수치 불안정이 증가하므로 단계적 도입이 필수다.
