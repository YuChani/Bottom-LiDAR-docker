# GMM-NDT 벤치마크 결과

**작성일**: 2026-03-16  
**데이터**: KITTI-style 7프레임 LiDAR 시퀀스  
**설정**: `--headless --window 4`, Sparse graph (chain), LM 최적화, 8 threads

---

## 1. 실험 개요

GMM-NDT (Gaussian Mixture Model + Normal Distribution Transform) factor를 기존 NDT와 비교하는 벤치마크 실험.

### 구성
- **NDT**: 기존 `IntegratedNDTFactor`, DIRECT7, ε=1e-3
- **GMM_NDT(K=1)**: `IntegratedGMMNDTFactor`, K=1 (단일 component), DIRECT7, ε=1e-3
- **GMM_NDT(K=2)**: `IntegratedGMMNDTFactor`, K=2 (2-component mixture), DIRECT7, ε=1e-3

## 6. 적용된 최적화

최초 구현 시 AC-3 시간 오버헤드가 1.82x로 미달이었으나, 아래 최적화를 적용하여 1.215x로 개선.

### 6.1 nth_element + partial sort (`update_correspondences`)

**변경 전**: 전체 후보를 `std::sort`로 정렬 후 top-K 선택  
**변경 후**: `std::nth_element`로 top-K만 파티셔닝 후, K개만 `std::sort` (O(N) → O(N + K log K))

### 6.2 Mahalanobis ratio pruning (`update_correspondences`)

상위 K개 후보 중, Mahalanobis 거리가 1위 후보의 `mahalanobis_ratio_threshold`배(기본 5.0)를 초과하는 후보는 제거. 정보가 적은 원거리 component 제거로 evaluate() 부하 감소.

### 6.3 Gamma hessian threshold (`evaluate`)

`gamma < gamma_hessian_threshold` (기본 0.05)인 component는 cost만 누적하고 Hessian 계산(J^T Ω J 등 6×6 행렬 연산)을 스킵. Cost 정확도는 유지하면서 가장 비싼 연산을 줄임.

### 6.4 Algebraic collapse (`evaluate`, K>1 전용)

**변경 전**: K개 component마다 `J^T * Ω_k * J` (4×6 · 4×4 · 4×6 = 6×6) 행렬 곱을 K번 수행  
**변경 후**:
- 4×4 누적: `M = Σ w_k * Ω_k`, 4-벡터 누적: `u = Σ w_k * Ω_k * r_k`
- 1회 행렬 곱: `J^T * M * J`, `J^T * u`
- 수학적으로 동일하나 행렬 곱 횟수 K→1로 감소

### 6.5 K=1 direct path (`evaluate`)

K=1일 때는 algebraic collapse를 거치지 않고, NDT의 evaluate()와 **완전히 동일한 연산 순서**로 직접 계산. 부동소수점 연산 순서가 동일하므로 AC-1 parity 보장.

## 7. 수정 이력

### `--gmm-components` CLI 버그 수정

**문제**: `main.cpp`에서 `--gmm_components` CLI 인수가 파싱되지 않아, `set_num_components(2)`가 하드코딩. K=1 지정이 불가능했음.

**수정**: `--gmm-components <k>` CLI argument를 proper하게 구현. `gmm_num_components` 멤버 + setter 추가, factory에서 `factor->set_num_components(gmm_num_components)` 사용.

### Q5 버그 수정 (Oracle 검증에 의해 발견)

**문제**: `log_det_cov` 계산에서 4×4 homogeneous covariance의 determinant를 사용하여 D=4로 처리.
4번째 차원은 w=0인 homogeneous 좌표이므로 실제 문제 차원은 3.

**수정 전**:
```cpp
const double det_inv = std::max(inv_cov_cache[v].determinant(), 1e-300);
log_det_cov_cache[v] = -std::log(det_inv) + 4.0 * std::log(2.0 * M_PI);
```

**수정 후**:
```cpp
const Eigen::Matrix3d inv_cov_3x3 = inv_cov_cache[v].block<3, 3>(0, 0);
const double det_inv_3x3 = std::max(inv_cov_3x3.determinant(), 1e-300);
log_det_cov_cache[v] = -std::log(det_inv_3x3) + 3.0 * std::log(2.0 * M_PI);
```

**이유**: 3×3 top-left block만 실제 3D 공간 정보를 담고 있으므로, D=3으로 Gaussian normalization 상수 계산.

## 8. 향후 개선 방향

1. **더 큰 데이터셋에서 검증**:
   - 현재 7프레임은 작은 규모. 더 긴 시퀀스에서 GMM의 이점이 더 드러날 수 있음
   - 환경 복잡도가 높은 (구조물 밀집, 좁은 통로) 시나리오에서 multi-modal matching의 효과 검증

2. **Adaptive K 스킴**:
   - 초기 iteration에서만 K=2, 수렴 근처에서 K=1로 전환하는 adaptive scheme
   - 지역 기하학 복잡도에 따라 voxel별로 K 자동 선택

3. **Cost/Responsibility 일관성**:
   - 현재: Responsibility는 Gaussian likelihood, Cost는 NDT surrogate (Option A)
   - Oracle 분석에 따르면 이론적으로는 불일치이나 실용적으로 작동
   - 필요시 true GMM NLL (Option C)로 전환 가능
