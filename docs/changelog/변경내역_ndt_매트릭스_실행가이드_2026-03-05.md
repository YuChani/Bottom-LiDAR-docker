# 변경내역: NDT 매트릭스 실험 반영 + 최종 고정값 정책 (2026-03-05)

## 1) 배경

- NDT 속도 개선을 위해 일시적으로 NDT 전용 파라미터(`search_mode`, `outlier_ratio`)를 외부에서 조절하며 매트릭스 실험을 수행했다.
- 이후 비교 공정성 관점에서 "NDT만 따로 조절하지 말고, 검증된 값으로 고정"하기로 결정했다.

## 2) 최종 정책 (현재 코드)

- NDT 전용 조절 UI/CLI는 제거
- NDT/LightNDT는 아래 값으로 내부 고정
  - `NDT search mode = DIRECT7`
  - `NDT outlier ratio = 0.01` (NDT만 해당)
- 공통 파라미터(모든 factor에 공통 적용)는 유지
  - `threads`
  - `correspondence_update_tolerance(rot/trans)`
    - 기본값: `rot=0.005`, `trans=0.02` (개선 실험 결과 반영)

## 3) 코드 변경 상세

### `src/main.cpp`

- 제거됨 (NDT 전용 오버라이드)
  - `--ndt-search-mode`
  - `--ndt-outlier-ratio`
  - GUI `NDT search mode` 콤보
  - GUI `NDT outlier ratio` 드래그
  - 관련 setter/파서/변환 유틸

- 유지됨 (공통 오버라이드)
  - `--factor`
  - `--threads`
  - `--corr-rot`
  - `--corr-trans`

- 공통 tolerance 기본값 변경
  - `correspondence_update_tolerance_rot: 0.0 -> 0.005`
  - `correspondence_update_tolerance_trans: 0.0 -> 0.02`

- NDT/LightNDT 생성 시 고정값 적용
  - NDT: `set_search_mode(DIRECT7)`, `set_outlier_ratio(0.01)`
  - LightNDT: `set_search_mode(DIRECT7)`

- 로그에 고정값 명시
  - `NDT Search Mode: DIRECT7 (fixed)`
  - `NDT Outlier Ratio: 0.010000 (fixed)`

- 부가 유지 사항
  - headless 종료 안정화(`std::_Exit(0)`)는 headless 모드에만 적용
  - 데이터 경로 폴백(`/root/workdir/data/pcd` -> `./data/pcd`) 유지

### `CMakeLists.txt`

- `Iridescence::Iridescence` 링크를 조건부로 유지
  - target이 있을 때만 링크하여 로컬 headless 빌드 호환성 확보

## 4) 실행 방법 (현재 기준)

## 4.1 빌드

```bash
cd /home/chani/personal/Bottom-LiDAR-docker
cmake -S . -B build_local
cmake --build build_local -j$(nproc)
```

## 4.2 옵션 확인

```bash
./build_local/lidar_registration_benchmark --help
```

현재 help:

```text
Usage: ./lidar_registration_benchmark [--headless] [--factor <name>] [--threads <n>] [--corr-rot <rad>] [--corr-trans <m>]
```

## 4.3 NDT 단일 실행

```bash
./build_local/lidar_registration_benchmark \
  --headless \
  --factor NDT \
  --threads 12 \
  --corr-rot 0.005 \
  --corr-trans 0.02
```

확인 로그:
- `Running: NDT`
- `NDT Search Mode: DIRECT7 (fixed)`
- `NDT Outlier Ratio: 0.010000 (fixed)`
- `Mean Translation Error`, `Mean Rotation Error`, `Optimization Time`

## 4.4 GUI(시각화) 실행

```bash
./build_local/lidar_registration_benchmark
```

GUI에서 변경 가능한 항목:
- `factor type`
- `optimizer type`
- `num threads`
- `corr update tolerance rot/trans`

GUI에서 변경 불가(고정):
- NDT search mode
- NDT outlier ratio
